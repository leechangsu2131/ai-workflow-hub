# 새 프로젝트 기획 워크플로우

1. 사용자가 프로젝트 목표와 배경을 설명합니다
2. 현재 `projects/` 폴더 안 기존 프로젝트들을 분석하여 중복/유사 여부 확인
3. 프로젝트를 작업 단위(이슈)로 쪼개기:
   - 각 이슈는 "무엇을 만들거나 고치는지" 명확히
   - 수락 기준(완료 조건) 최소 2개 포함
   - 담당 도구(Cursor/Codex/Antigravity) 지정
4. `docs/plans/` 에 플랜 문서 작성 후 push
5. 사용자에게 이슈 목록 검토 요청
6. 승인 후 GitHub Issues로 등록 안내
